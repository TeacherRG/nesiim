# CLAUDE.md — Проект Несиим (נְשִׂיאִים)

Этот файл содержит информацию о структуре кодовой базы, архитектурных решениях и рабочих процессах разработки для проекта **Несиим** — интерактивной платформы изучения Торы.

---

## О проекте

**Несиим** (ивр. נְשִׂיאִים — «вожди/князья») — одностраничное веб-приложение для 13-дневного плана чтения текстов из книги Числа (Бемидбар), глав 7–8. Посвящено приношениям 12 вождей колен Израиля при освящении Скинии, и предназначено для периода месяца Нисан (2026 г. / 5786 г.).

- **Живой сайт:** [nesiim.mychitas.app](https://nesiim.mychitas.app)
- **Хостинг:** GitHub Pages
- **Язык интерфейса:** русский (по умолчанию), немецкий, иврит
- **Автор:** TeacherRG (kosherletter@gmail.com)

## Архитектура

### Подход

Статическое SPA с прогрессивным улучшением, без зависимостей. Всё приложение содержится в трёх файлах JavaScript/HTML. Не используются фреймворки, бандлеры или пакетные менеджеры.

### index.html (808 строк)

Основной файл приложения. Содержит разметку, встроенные стили и логику JavaScript в одном файле.

**Структура HTML:**
- Строки 1–93: `<head>` — подключение шрифтов (Google Fonts), метатеги, встроенные стили
- Строки 94–462: `<body>` — структура контента
- Строки 465–752: Основная логика JavaScript
- Строки 754–806: Инициализация уведомлений

**Ключевые компоненты интерфейса:**

| Компонент | ID / Класс | Описание |
|-----------|-----------|----------|
| Навигация по дням | `#nav` | Закреплённая панель с 13 кнопками (День 1 – День 13) |
| Настройки | `#settings-panel` | Язык, размер шрифта, выбор шрифта |
| Прогресс | `#progress-bar` | Полоса прогресса (0–13 прочитанных дней) |
| Секции дней | `#d1` – `#d13` | Контент для каждого дня чтения |
| Счётчик посетителей | `#visit-count` | Получает данные с CounterAPI |

**Структура секции дня:**
```
div.day#d{N}
  ├── Заголовок (еврейское имя, транслитерация, колено, ссылка на Тору)
  ├── div.parallel-text (CSS Grid: 2 колонки для ру/де, 1 для иврита)
  │     ├── div.hebrew-text (RTL, шрифт Noto Serif Hebrew)
  │     └── div.translation-text (data-lang="ru|de")
  ├── div.prayer (молитва יְהִי רָצוֹן с переводами)
  └── button.read-btn (отметить день как прочитанный)
```

### Система локализации

Реализована без сторонних библиотек:

- HTML-элементы используют атрибут `data-lang="ru|de|he"` для управления видимостью
- Переключение языка обновляет DOM-класс `<html>` (`.lang-ru`, `.lang-de`, `.lang-he`)
- CSS-правила скрывают/показывают элементы на основе текущего класса языка
- Класс `.lang-he` активирует однолонный RTL-макет вместо двухколоночного
- Текущий язык сохраняется в `localStorage['nesiim-lang']`

**Поддерживаемые языки:** `'ru'` (русский), `'de'` (немецкий), `'he'` (иврит)

**Определение языка при загрузке:**
1. Проверяется `localStorage['nesiim-lang']`
2. Если не задан — используется язык браузера (`navigator.language`)
3. По умолчанию: русский

### Хранение состояния (localStorage)

| Ключ | Тип | Описание |
|------|-----|----------|
| `nesiim-read` | JSON-объект | `{"1": true, "2": true, ...}` — прочитанные дни |
| `nesiim-lang` | строка | Текущий язык: `'ru'`, `'de'` или `'he'` |
| `nesiim-fs` | строка | Размер шрифта в процентах (по умолчанию `"100"`) |
| `nesiim-font` | строка | Ключ выбранного шрифта из объекта `FONTS` |
| `nesiim-notif` | строка | Статус уведомлений: `'granted'`, `'denied'`, `'dismissed'` |

`sessionStorage['nesiim-visited']` — флаг первого визита в рамках сессии (используется для логики уведомлений).

### Навигация по дням

- 13 секций (`#d1` – `#d13`) изначально скрыты через CSS
- Активная секция показывается добавлением класса `.on`
- При загрузке страницы автоматически вычисляется текущий день: отсчёт от 1 Нисана (19 марта 2026 г.)
- Нажатие кнопки навигации скрывает текущую секцию и показывает выбранную

### notifications.js (157 строк)

ES6-модуль для управления push-уведомлениями браузера.

**Ключевые функции:**

| Функция | Описание |
|---------|----------|
| `isInActiveWindow()` | Проверяет, попадает ли текущая дата в период 1–13 Нисана (19–31 марта 2026) |
| `getNext10am()` | Вычисляет следующее 10:00 утра (пропускает субботы) |
| `registerSW()` | Регистрирует Service Worker |
| `scheduleNext(lang)` | Отправляет сообщение SW для планирования следующего уведомления |
| `shouldShowBanner()` | Определяет, нужно ли показывать баннер запроса разрешения |
| `requestAndSchedule(lang)` | Запрашивает разрешение и планирует первое уведомление |
| `initNotifications(lang)` | Перепланирует уведомление при загрузке (если уже разрешено) |

**Логика уведомлений:**
- Активны только в период 1–13 Нисана (19–31 марта 2026)
- Ежедневно в 10:00, кроме суббот
- Разрешение запрашивается через баннер (не браузерный диалог напрямую)
- Статус сохраняется в `localStorage['nesiim-notif']`

### sw.js (52 строки)

Service Worker для планирования и отображения уведомлений.

**Обработчики событий:**

| Событие | Действие |
|---------|---------|
| `install` | Немедленная активация через `skipWaiting()` |
| `activate` | Захват всех клиентов через `clients.claim()` |
| `message` | Получает `{ type: 'SCHEDULE', delay, lang, activeEnd }`, планирует уведомление |
| `notificationclick` | Открывает/фокусирует сайт |

**Параметры уведомления:**
- Тег: `'nesiim-daily'` (заменяет предыдущее уведомление)
- Иконка: `/favicon.ico`
- Заголовок и текст: локализованы на основе `lang`

## Рабочие процессы разработки

### Локальный запуск

Нет системы сборки — просто откройте файл в браузере:

```bash
# Вариант 1: Python (рекомендуется для тестирования уведомлений — требуется HTTP)
python3 -m http.server 8000
# Затем: http://localhost:8000

# Вариант 2: Node.js
npx http-server -p 8000

# Вариант 3: Напрямую (push-уведомления работать не будут)
open index.html
```

> **Важно:** Push-уведомления и Service Worker требуют HTTPS или `localhost`. При открытии через `file://` они недоступны.

### Добавление/изменение контента

**Изменение текста перевода:**
- Найдите нужную секцию дня (`#d1` – `#d13`) в `index.html`
- Элементы с `data-lang="ru"` — русский текст
- Элементы с `data-lang="de"` — немецкий текст
- Элементы внутри `div.hebrew-text` — текст на иврите

**Изменение молитв (יְהִי רָצוֹן):**
- Молитвы находятся в блоках `div.prayer` внутри каждой секции дня
- Каждый перевод обёрнут в `<span data-lang="ru|de|he">`

**Добавление нового шрифта:**
- Добавьте шрифт в объект `FONTS` в JS-секции `index.html`
- Добавьте CSS-правило `@font-face` или импорт Google Fonts
- Шрифт автоматически появится в панели настроек

### Деплой

Деплой производится автоматически через GitHub Pages:

1. Внесите изменения в `index.html`, `notifications.js` или `sw.js`
2. Сделайте commit и push в ветку `main`
3. GitHub Pages автоматически опубликует изменения (обычно за 1–2 минуты)
4. Сайт доступен по адресу `nesiim.mychitas.app`

```bash
git add index.html notifications.js sw.js
git commit -m "Описание изменений"
git push origin main
```

> **Примечание:** Файл `CNAME` (`nesiim.mychitas.app`) нельзя удалять — он необходим для привязки домена к GitHub Pages.

## Важные архитектурные решения

1. **Нет зависимостей** — намеренный выбор для максимальной простоты и долгосрочной поддержки без устаревших пакетов.

2. **Весь контент в одном HTML-файле** — упрощает хостинг и устраняет необходимость в сборке. Компромисс: большой размер файла (~808 строк), но приемлемо для данного масштаба.

3. **localStorage вместо сервера** — прогресс пользователя хранится локально. Нет регистрации, нет данных на сервере.

4. **Service Worker для уведомлений** — единственная причина наличия SW. Не используется для кэширования или офлайн-режима.

5. **Период активности уведомлений захардкожен** — активны только с 19 по 31 марта 2026 г. (1–13 Нисана 5786). Это сознательное решение, так как приложение приурочено к конкретному событию.
