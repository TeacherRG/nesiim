# נְשִׂיאִים — Насии

Интерактивная платформа для изучения Торы, посвящённая отрывку **Бемидбар (Числа) 7–8** — приношениям двенадцати вождей колен Израилевых при освящении Скинии. Проект рассчитан на **месяц Нисан 5786** (19–31 марта 2026 г.).

🔗 **[nesiim.mychitas.app](https://nesiim.mychitas.app)**

---

## О проекте

Каждый день месяца Нисан один из вождей (насий) приносил одинаковое жертвоприношение в знак посвящения жертвенника. Несмотря на то что все приношения были идентичны, Тора описывает каждое из них отдельно — чтобы подчеркнуть неповторимость намерения и вклада каждого человека.

Платформа предоставляет **13-дневный план чтения** (12 вождей + итоговый день) с еврейским оригиналом и переводами, удобной навигацией и отслеживанием прогресса.

---

## Возможности

- 📅 **13-дневный план** — с 1 по 13 нисана, с автоматическим переходом к текущему дню
- 🌐 **Три языка** — русский (по умолчанию), немецкий и иврит
- 📖 **Параллельные тексты** — оригинал на иврите рядом с переводом
- ✅ **Отслеживание прогресса** — отметка о прочтении, прогресс-бар (0/13)
- 🔄 **Сброс прогресса** — кнопка для обнуления всех отметок
- 💾 **Сохранение данных** — прогресс сохраняется в `localStorage` браузера
- 🔔 **Push-уведомления** — ежедневное напоминание в 10:00 (кроме субботы) в течение активного периода
- 🕍 **Молитва** — раздел «יְהִי רָצוֹן» (Йеhи рацон) для каждого из 12 дней
- 🔤 **Настройка отображения** — выбор шрифта (6 вариантов), размер текста (80–150%)

## Архитектура

Проект реализован как **одностраничное статическое приложение** без зависимостей и этапа сборки.

### `index.html`

Содержит всё приложение в одном файле: HTML-структуру, встроенные стили (CSS) и логику (JavaScript).

**Структура страницы:**
- Заголовок с многоязычным названием
- Навигационная панель с 13 кнопками (12 вождей + итоговый день)
- Фиксированная панель с языковым переключателем и настройками
- Прогресс-бар и кнопка сброса
- Баннер согласия на уведомления
- 13 секций с текстом: иврит + перевод построчно, молитва

**Ключевые JS-функции:**

| Функция | Описание |
|---|---|
| `show(day)` | Переключение между днями |
| `toggleRead(day)` | Отметить/снять отметку о прочтении |
| `resetProgress()` | Сбросить весь прогресс |
| `applyFontSize(value)` | Изменить размер шрифта (80–150%) |
| `applyFont(font)` | Сменить шрифт (6 вариантов) |
| `setLang(lang)` | Переключить язык (ru / de / he) |
| `toggleSettings(event)` | Открыть/закрыть панель настроек |

**Поддерживаемые шрифты:**
- Frank Ruhl Libre *(по умолчанию)*
- Noto Serif Hebrew
- Crimson Pro
- Stam Ashkenaz CLM *(локальный TTF)*
- Times New Roman
- David Libre

### `notifications.js`

Управляет системой push-уведомлений:

- **Активный период:** 19–31 марта 2026 г. (1–13 Нисана 5786)
- **Расписание:** ежедневно в 10:00, суббота пропускается
- Запрашивает разрешение браузера при согласии пользователя
- Регистрирует Service Worker и передаёт ему задачу планирования
- Поддерживает многоязычные сообщения (ru / de / he)
- Сохраняет решение пользователя в `localStorage`

**Ключевые функции:**

| Функция | Описание |
|---|---|
| `initNotifications(lang)` | Инициализация при загрузке страницы |
| `shouldShowBanner()` | Показывать ли баннер согласия |
| `dismissBanner()` | Сохранить отказ пользователя |
| `requestAndSchedule(lang)` | Запросить разрешение и запланировать |
| `getNext10am()` | Вычислить следующее время 10:00 (без субботы) |
| `scheduleNext(lang)` | Отправить задачу в Service Worker |

### `sw.js`

Service Worker — работает в фоне:

- Обрабатывает сообщения `SCHEDULE` от основного потока
- Откладывает отображение уведомления на рассчитанное время
- Показывает локализованное уведомление (ru / de / he)
- При нажатии на уведомление фокусирует вкладку или открывает новую

## Запуск локально

Установка зависимостей и сборка не требуются:

```bash
# Открыть в браузере напрямую
open index.html

# Или запустить простой HTTP-сервер (для Service Worker):
python3 -m http.server 8080
# Затем открыть http://localhost:8080
```

> **Важно:** Service Worker и push-уведомления работают только через `http://localhost` или `https://`. При открытии через `file://` уведомления будут недоступны.
